1. Brand Overview

Brand Name: Love Darts
Product: Social darts scoring and statistics application
Theme: Modern rustic pub darts culture

The product should feel:

Warm, social, and authentic

Competitive but friendly

Traditional with modern polish

Avoid anything that feels:

Neon, arcade-like, or flashy

Corporate or sterile

Cartoonish or novelty-driven

2. Brand Personality & Tone

The brand voice is:

Clear and human

Friendly but not jokey

Confident, not salesy

UI copy must be:

Short

Direct

Functional

Avoid:

Buzzwords

Over-explaining

Clever wording that harms clarity

3. Typography (Strict)

Primary and only font: Sansation

Sansation must be used throughout the entire application, including:

Headings

Body text

Buttons

Navigation

Forms

Labels

Stats

Modals

Tooltips

Allowed weights:

Regular

Bold

Rules:

No other fonts may be introduced

No decorative effects (shadows, outlines, gradients)

No novelty, serif, script, or display fonts

Fallback fonts are for technical failure only (e.g. Arial, system-ui)

6. Buttons & Actions (Critical)
6.1 Primary Buttons

Primary action buttons must be red.

Approved primary button red:

#350D14

Used for:

Start Game

Save

Confirm

Submit

Rules:

Only one primary action per screen where possible

Red must never be used for navigation or passive actions

6.2 Secondary Buttons

Use brown tints or tones

Never red

Must visually defer to primary buttons

6.3 Button States

Hover: subtle darkening or saturation shift

Active: darker shade of same colour

Disabled: muted brown tone, never red

No glow, bounce, or exaggerated animation

7. Layout & UI Design

Principles:

Clean, structured layouts

Strong visual hierarchy

Generous spacing

Solid, tactile-feeling cards

Cards:

Entire card clickable where applicable

Subtle hover and active states

No exaggerated elevation

Buttons:

Weighty and grounded

No pill-style buttons unless explicitly requested

8. Interaction & Motion

Minimal, purpose-driven animation

Smooth transitions only

Motion must provide feedback, not decoration

Avoid:

Bouncy effects

Flashy transitions

Gimmicks

9. Accessibility (Non-Negotiable)

Must meet WCAG contrast requirements

Keyboard navigation supported

If accessibility conflicts with aesthetics, accessibility wins
